SECTION 02

SYSTEM ARCHITECTURE

FRONTEND CLIENT
  • React 19 & Vite: High-performance single page client app.
  • Context API: Manage active user session variables and auth tokens.
  • Axios Client: Interceptors handling server API headers and token refreshing.
  • Socket.IO Client: Persistent WebSocket handshakes.
  • CSS Tokens: Premium dark theme and glassmorphic designs.
BACKEND API SERVER
  • Node.js & Express: Asynchronous HTTP routing, file handling, and API gateways.
  • Socket.IO Server: Manages real-time rooms, events emitter, and client states.
  • JWT & bcryptjs: User authorization and secure bcrypt password hashing.
  • Sequelize ORM: Manages schemas, relations, and auto-seeding protocols.
DATABASE ENGINE
  • MySQL Server: Standard relational database storage.
  • Configured Constraints: Cascading foreign key definitions ensuring relational integrity.
  • Localized Storage: Dedicated `mysql_data` folder for portable startup configurations.
  • Automatic Syncing: Backend server auto-creates schemas and seeds baseline system tables.

SECTION 04

REAL-TIME NOTIFICATION SUBSYSTEM

SOCKET ROOM STRATEGY
  • Direct User Room (`user_`): Dedicated room mapped to unique user database ID. Handles highly targeted alert notifications (e.g. status revisions for the citizen, new casework assigned to the officer).
  • Role-Based Rooms: Global listeners for role broadcast groups:
    • `admin` room - Broadcasts new complaints.
    • `officer` room - Broadcasts updates.
    • `citizen` room - Broadcasts system alerts.
  • Client Syncing: Frontend state context hooks onto socket listeners, prompting immediate state refetches.
EVENT TRANSITIONS:
SUBMIT Citizen files Case COMP-0231. Server writes to DB & emits "notification" payload to "admin" room.
ASSIGN Admin delegates case to Officer. Server emits "notification" payload to room "user_2".
RESOLVE Officer marks case RESOLVED. Server emits "notification" payload to room "user_5". Toast alert triggers on citizen client.

SECTION 09

SECURITY & DATA INTEGRITY

AUTHENTICATION & SESSIONS
  • Password Protection: Employs `bcryptjs` hashing with automated salt generation. Raw plain-text passwords are never logged or stored.
  • Stateless Access Tokens: JWT tokens issued upon successful login, authorizing frontend Axios requests using authorization headers.
  • Auth Guard Middleware: Express backend routes run access-token checking middleware. React client routes protect directories via context validation.
INTEGRITY & PROTECTION
  • SQL Injection Blocks: Enforces strict data-mapping constraints via Sequelize parameterized queries, blocking database attacks.
  • Cross-Origin Safeguards: Employs CORS middleware security configurations, locking server resources to recognized frontend clients.
  • Upload Directories Lockdown: Evidence uploader routes validate file sizes and file extensions, protecting server space from scripts.
SECTION 10

SETUP & EXECUTION INSTANCE

WINDOWS AUTOMATIC BATCH RUNNER

The system includes a pre-configured `start-services.bat` launcher that simplifies deployment:

  • Task 1: Checks local environments and installs Node modules for root & backend.
  • Task 2: Provisions localized database configurations inside the `mysql_data` directory.
  • Task 3: Launches the MySQL Server instance (`mysqld.exe`) on port `3306`.
  • Task 4: Starts Node API (`localhost:8080`) and Vite Client (`localhost:5173`) and opens the browser.
MANUAL CONSOLE COMMANDS
  • 1. Install packages: npm install && cd backend && npm install
  • 2. Configure properties: Setup MySQL host, user, and password in backend/.env
  • 3. Start Database: "mysqld.exe" --datadir="D:\cyber security\mysql_data" --port=3306
  • 4. Launch Backend API: cd backend && npm run dev (starts DB migrations & seeding)
  • 5. Launch Frontend App: npm run dev (opens app on localhost:5173)
